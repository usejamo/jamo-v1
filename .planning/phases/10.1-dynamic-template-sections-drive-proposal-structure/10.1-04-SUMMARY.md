# Phase 10.1 Plan 04: UI Wiring — Dynamic Sections + Template Badge — Summary

**One-liner:** ProposalDetail now fetches proposal_sections with id/name/position ordered by position and renders a "Template: [name]" badge above SectionWorkspace; ProposalsList fetches and shows the template name on each proposal card.

## What Was Built

**Task 1 — ProposalDetail.tsx:**
- Extended `proposalSections` state type to include `id: string`, `name: string | null`, `position: number | null`
- Both the initial `useEffect` fetch and `refetchSections` callback now select `id, section_key, name, position, content, is_locked, status, last_saved_content, compliance_flags` and chain `.order('position', { ascending: true })`
- Added `templateName` state and a `useEffect` that fetches `templates.name` by `selected_template_id` when proposal loads
- Added template badge JSX `<div>Template: <span>{templateName}</span></div>` above `<SectionWorkspace />` inside the `!isStreamingMode && generated && sectionsLoaded` block (D-13)
- Extended `sections` map passed to SectionWorkspace to include `id`, `name`, `position` fields

**Task 2 — ProposalsList.tsx:**
- Added `supabase` import from `../lib/supabase`
- Added `templateNames: Record<string, string>` state
- Added `useEffect` that collects unique `selected_template_id` values from proposals, fetches `templates(id, name)` via `.in()`, and populates the map
- Renders template name as `<span className="text-xs text-gray-400">` below the TA/studyType line on each proposal card (D-15)
