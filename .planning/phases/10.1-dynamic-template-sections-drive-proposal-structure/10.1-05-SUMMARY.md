# Phase 10.1 Plan 05: Template Extraction Pipeline + Role Editing UI — Summary

**One-liner:** template-extract edge function now captures section descriptions from following content and calls Claude Haiku to classify roles into a validated allowlist; TemplatesTab exposes per-section role dropdowns; Step4Generate pre-selects the default template on mount.

## What Was Built

### Task 1 — template-extract edge function (D-06)

Three targeted changes to `supabase/functions/template-extract/index.ts`:

1. **`parseSectionsFromHtml`** refactored from `string[]` to `Array<{ name: string; description: string | null }>`. Uses `html.split(/<h[1-3]...>)` to pair each heading with its following content block; description truncated at 300 chars.

2. **`parseSectionsFromText`** similarly updated. Captures up to 5 following non-heading lines as description (joined, 300-char max); stops at next ALL_CAPS heading heuristic match.

3. **`classifyRoles`** new async function: sends section names to `claude-haiku-4-5-20251001` with a structured prompt; parses JSON response; validates each returned role against `KNOWN_ROLES` allowlist before storing — unknown/hallucinated roles become `null`. Returns empty map on any error (graceful degradation).

4. **Main handler** wired: `sections` now typed as objects; `classifyRoles` called after parsing if `ANTHROPIC_API_KEY` is set; `sectionInserts` includes both `description` and `role` columns.

### Task 2 — TemplatesTab + Step4Generate

**TemplatesTab.tsx:**
- `handleRoleChange(sectionId, newRole)` added inside `SectionDisclosure`: calls `supabase.from('template_sections').update({ role })` then updates local state optimistically.
- Each section list item now renders a `<select>` with 9 known-role options + "None" (value="", maps to null). LLM-classified role pre-fills the dropdown.

**Step4Generate.tsx:**
- `Template` interface gains `is_default: boolean`.
- Select query updated: `.select('id, name, description, source, parse_status, is_default')`.
- After fetch: if `state.selectedTemplateId` is null, finds `is_default === true` template and dispatches `SET_TEMPLATE` — implements D-16.
- `ContextSummary` template line simplified: `templateLabel ?? 'Standard Proposal'` — never shows blank or "No template" string.
