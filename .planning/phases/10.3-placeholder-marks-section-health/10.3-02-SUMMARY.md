## Changes Made

### SectionEditorBlock.tsx (Task 1 — previous commit)
- Imports `migratePlaceholders` and `PlaceholderMark`
- Runs `migratePlaceholders(rawContent)` before `useEditor`; uses `migratedContent` as editor `content:`
- `PlaceholderMark` added to TipTap extensions array
- Migration autosave `useEffect` calls `saveNow(migratedContent)` on mount if content changed
- Analyzer `useEffect` with 200ms debounce walks doc, collects placeholder marks, dispatches `UPDATE_SECTION_ISSUES`

### SectionNavPanel.tsx (Task 2)
- `resolveStatus` now computes `const hasIssues = Object.values(editorState.issues).some(list => list.length > 0)` and `const hasLegacyFlags = editorState.compliance_flags.length > 0`; returns `'needs-review'` when either is true
- JSX render loop extended: each issue renders a `<span>` with `displayLabel` — placeholder category prefixed with `"Missing: "`, others use `issue.label` directly

### generate-proposal-section/index.ts (Task 3)
- `escapeHtmlInline` inlined with SYNC OBLIGATION comment (byte-for-byte identical to `src/lib/escapeHtml.ts`)
- `placeholderPatternToSpan` inlined from `src/lib/placeholderHtml.ts`
- `flush()` adds post-process pass: `fullText.replace(/\[PLACEHOLDER:\s*([^\]]+)\]/g, ...)` assigns to `processedText`; both `writeSectionById` and `writeSection` receive `processedText` instead of `fullText`
