## What Was Built

All Wave 1 foundations for the placeholder marks system:

1. **workspace.ts** â€” `IssueCategory` union, `SectionIssue` interface, `issues` field on `SectionEditorState`, `UPDATE_SECTION_ISSUES` action in `WorkspaceAction` union.

2. **SectionWorkspaceContext.tsx** â€” `UPDATE_SECTION_ISSUES` reducer case that spreads `section.issues` and replaces the given category array (one writer per category semantics).

3. **src/lib/escapeHtml.ts** â€” XSS guard escaping `&`, `<`, `>`, `"` before embedding label in HTML attributes.

4. **src/lib/placeholderHtml.ts** â€” `placeholderPatternToSpan(label, id)` shared helper used by both the edge function post-process pass and `migratePlaceholders`. Single source of truth for the `<span data-placeholder-id="..." data-placeholder-label="...">` HTML format.

5. **src/lib/migratePlaceholders.ts** â€” `migratePlaceholders(html)` regex replace of `[PLACEHOLDER: label]` patterns to span elements with fresh UUIDs. Becomes a no-op after first autosave.

6. **src/components/editor/extensions/PlaceholderMark.ts** â€” TipTap `Mark.create` extension with `name: 'placeholder'`, `inclusive: false`, `spanning: true`. Parses `span[data-placeholder-id]`, renders amber pill (`bg-amber-100 text-amber-800 rounded px-0.5`). Resolution plugin (`appendTransaction`) removes the mark when edited text is detected, with try/catch for partial resolution on coordinate mismatches.
