# Phase 10.3: Placeholder Marks & Section Health System — Context

**Gathered:** 2026-04-28
**Status:** Ready for planning
**Source:** PRD Express Path (docs/superpowers/specs/2026-04-28-placeholder-section-health-design.md)

<domain>
## Phase Boundary

This phase replaces the fragile `[PLACEHOLDER: ...]` string-based detection with a persistent TipTap custom mark. It adds a generalized `issues` system to `SectionEditorState` driven by `UPDATE_SECTION_ISSUES` dispatches. The status dot in `SectionNavPanel` reads from both the new `issues` field and the existing `compliance_flags` bridge (bridge is NOT removed in this phase).

**NOT in scope:** compliance migration to `UPDATE_SECTION_ISSUES`, deterministic analyzer IDs, pending/ready per-category status shape (all deferred to §8 of spec — documented there for future phases).

</domain>

<decisions>
## Implementation Decisions

### Types (workspace.ts)

- Add `IssueCategory = 'placeholder' | 'compliance' | 'typo' | 'cross-section'` union type
- Add `SectionIssue` interface: `{ id: string; label: string; message?: string }`
  - `label` MUST be non-empty, raw/clean with no UI prefix (e.g. `"sponsor's full legal name"`, never `"Missing: sponsor's full legal name"`)
  - Display prefixes (e.g. `"Missing: "`) are added in the render layer, not stored in `label`
- Add `issues: Partial<Record<IssueCategory, SectionIssue[]>>` to `SectionEditorState`
  - Initialized as `{}` on mount
  - A missing category is treated as "no issues" (V1 absent-category contract)
- Add workspace action: `{ type: 'UPDATE_SECTION_ISSUES'; payload: { section_key: string; category: IssueCategory; issues: SectionIssue[] } }`
  - Semantics: replaces full list for the given category (one writer per category, no partial patches)
  - A dispatch with `issues: []` clears the category

### Reducer (SectionWorkspaceContext.tsx)

- Add reducer case for `UPDATE_SECTION_ISSUES`: sets `state.sections[section_key].issues[category] = issues`

### Escape helper (src/lib/escapeHtml.ts) — NEW FILE

```typescript
export function escapeHtml(s: string): string {
  return s
    .replace(/&/g, '&amp;')
    .replace(/</g, '&lt;')
    .replace(/>/g, '&gt;')
    .replace(/"/g, '&quot;')
}
```

### Shared conversion helper (src/lib/placeholderHtml.ts) — NEW FILE

```typescript
import { escapeHtml } from './escapeHtml'

export function placeholderPatternToSpan(label: string, id: string): string {
  const escaped = escapeHtml(label.trim())
  return `<span data-placeholder-id="${id}" data-placeholder-label="${escaped}">${escaped}</span>`
}
```

Used by BOTH the edge function post-process pass and `migratePlaceholders`. Must not diverge.

### Legacy backfill (src/lib/migratePlaceholders.ts) — NEW FILE

```typescript
import { placeholderPatternToSpan } from './placeholderHtml'

export function migratePlaceholders(html: string): string {
  return html.replace(
    /\[PLACEHOLDER:\s*([^\]]+)\]/g,
    (_, raw) => placeholderPatternToSpan(raw, crypto.randomUUID())
  )
}
```

Applied in `SectionEditorBlock` before content is passed to `useEditor`. If `migratedContent !== rawContent`, call `saveNow(migratedContent)` immediately to commit UUIDs to DB and prevent remount UUID drift. This function becomes a no-op for that row on all future loads once the autosave completes.

### TipTap PlaceholderMark (src/components/editor/extensions/PlaceholderMark.ts) — NEW FILE

- `Mark.create({ name: 'placeholder', spanning: true, inclusive: false })`
- Attributes: `id` (maps to/from `data-placeholder-id`), `label` (maps to/from `data-placeholder-label`)
- `parseHTML`: `[{ tag: 'span[data-placeholder-id]' }]`
- `renderHTML`: `['span', mergeAttributes(HTMLAttributes, { class: 'bg-amber-100 text-amber-800 rounded px-0.5' }), 0]`
- Display text inside mark is the label only — no brackets, no prefix
- `inclusive: false` means typing at boundary does not extend the mark
- Resolution plugin (`resolutionPlugin(this.type)`) attached via `addProseMirrorPlugins()`

#### Resolution plugin behavior:
- On `appendTransaction`: if any transaction changed the doc, collect all placeholder mark IDs that were touched (via `mapping.maps` + `nodesBetween`)
- Call `collectPlaceholderRanges(doc, markType, touchedIds)` to get full contiguous ranges per ID
- Remove the mark from the full range for each touched ID
- Try/catch around `nodesBetween` to handle compound-transaction coordinate mismatches without crashing (partial resolution acceptable; crash is not)
- Dev-only: `console.warn` if non-contiguous mark ranges detected for the same ID

### SectionEditorBlock changes (src/components/editor/SectionEditorBlock.tsx)

1. Register `PlaceholderMark` in the TipTap extensions array alongside `StarterKit`, `Table`, etc. Confirm no other extension claims the name `'placeholder'`
2. Apply `migratePlaceholders(rawContent)` before passing to `useEditor`; if content changed, call `saveNow(migratedContent)` immediately
3. Add placeholder analyzer `useEffect` keyed on `[editor, dispatch, sectionKey]`:
   - Walk `editor.state.doc.descendants` collecting unique placeholder mark IDs → `SectionIssue[]`
   - Dispatch `UPDATE_SECTION_ISSUES` with `category: 'placeholder'`
   - Debounced at 200 ms on `editor.on('update', handler)`
   - Cleanup: `editor.off('update', handler); handler.cancel()`

### SectionNavPanel (src/components/editor/SectionNavPanel.tsx)

Update `resolveStatus` function:

```typescript
function resolveStatus(editorState: SectionEditorState): DotStatus {
  if (editorState.status === 'generating') return 'generating'
  if (editorState.status === 'error')      return 'error'
  if (!editorState.content)                return 'missing'

  const hasIssues      = Object.values(editorState.issues).some(list => list.length > 0)
  const hasLegacyFlags = editorState.compliance_flags.length > 0
  if (hasIssues || hasLegacyFlags) return 'needs-review'

  return 'complete'
}
```

When rendering a `placeholder` category issue in the nav panel, prepend `"Missing: "` as display prefix:
```typescript
const displayLabel = category === 'placeholder'
  ? `Missing: ${issue.label}`
  : issue.label
```

### Edge function post-process pass (supabase/functions/generate-proposal-section/index.ts)

After SSE stream completes and BEFORE writing `content` to `proposal_sections`:

```typescript
content = content.replace(
  /\[PLACEHOLDER:\s*([^\]]+)\]/g,
  (_, raw) => placeholderPatternToSpan(raw, crypto.randomUUID())
)
```

The edge function imports `placeholderPatternToSpan` from the shared helper (or inlines it — edge function environment must be able to access it). IDs assigned once at write time; stable on all subsequent loads via TipTap `parseHTML`.

Note: `crypto.randomUUID()` is available natively in Deno/Edge runtime.

### Integration test

Add a test exercising the full roundtrip:
1. Edge function post-process converts `[PLACEHOLDER: budget figure]` → `<span data-placeholder-id="uuid" data-placeholder-label="budget figure">budget figure</span>`
2. Content written to DB
3. TipTap loads content → `parseHTML` restores mark with same UUID
4. `getHTML()` → `renderHTML` produces span with same UUID
5. Assert step-3 UUID === step-4 UUID

This test also guards against markdown-layer interference treating `[...]` as a link.

<specifics>
## Specific Ideas

### Roundtrip HTML contract
```
[PLACEHOLDER: label]  (transport format — never shown in editor after deploy)
    ↓ edge function post-process OR migratePlaceholders()
<span data-placeholder-id="uuid" data-placeholder-label="label">label</span>
    ↓ TipTap parseHTML
placeholder mark { id: uuid, label: "label" }
    ↓ TipTap renderHTML / getHTML()
<span data-placeholder-id="uuid" data-placeholder-label="label" class="bg-amber-100 text-amber-800 rounded px-0.5">label</span>
    ↓ DB write (autosave)
same UUID preserved ✓ — remount is a no-op
```

### ID stability contract
- Placeholder IDs originate from edge function crypto.randomUUID() at generation time
- Preserved through getHTML() → DB → parseHTML() on every load
- `migratePlaceholders` generates UUIDs for legacy rows ONCE then autosaves immediately
- Analyzer-generated IDs for other categories MUST be deterministic (not crypto.randomUUID()) — but this is deferred to §8.2

### What NOT to build in this phase
- Compliance migration to UPDATE_SECTION_ISSUES (§8.1)
- Deterministic compliance IDs (§8.2)
- Pending/ready per-category status shape (§8.3)
- Remove compliance_flags bridge from resolveStatus

</specifics>
