**File 1: `src/hooks/useSectionAIAction.ts`**

Hook that triggers an AI action (Generate, Regenerate, Expand, Condense, Rewrite) for a specific section by calling the `generate-proposal-section` edge function via SSE streaming.

```typescript
import { useCallback } from 'react'
import { supabase } from '../lib/supabase'
import { useSectionWorkspace } from '../context/SectionWorkspaceContext'
import type { AIActionType } from '../types/workspace'
```

Export:
```typescript
export function useSectionAIAction(proposalId: string, sectionKey: string, orgId: string)
```

Returns: `{ triggerAction: (actionType: AIActionType, currentContent: string) => Promise<void> }`

`triggerAction` implementation:
1. **Snapshot before action (D-02):** Take `currentContent` parameter (the editor's current HTML). Write to `proposal_section_versions` via supabase:
   ```typescript
   await supabase.from('proposal_section_versions').insert({
     proposal_id: proposalId, org_id: orgId, section_key: sectionKey,
     content: currentContent, action_label: `Before ${actionType.charAt(0).toUpperCase() + actionType.slice(1)}`
   })
   ```
2. **Prune old versions:** Count versions for this section. If count > 20, delete oldest entries to keep at 20.
3. **Dispatch START_AI_ACTION** with `{ section_key: sectionKey, action_type: actionType, snapshot: currentContent }`.
4. **SSE streaming** — use the same raw `fetch()` pattern from `useProposalGeneration.ts`:
   ```typescript
   const url = `${import.meta.env.VITE_SUPABASE_URL}/functions/v1/generate-proposal-section`
   const { data: { session } } = await supabase.auth.getSession()
   const response = await fetch(url, {
     method: 'POST',
     headers: {
       'Content-Type': 'application/json',
       'Authorization': `Bearer ${session?.access_token}`,
       'apikey': import.meta.env.VITE_SUPABASE_PUBLISHABLE_KEY,
     },
     body: JSON.stringify({
       proposal_id: proposalId,
       section_key: sectionKey,
       action: actionType, // edge function uses this to modify prompt
       existing_content: actionType !== 'generate' ? currentContent : undefined,
     }),
   })
   ```
5. **Read SSE stream:** Parse SSE lines. On each text chunk, dispatch `UPDATE_AI_PREVIEW` with accumulated text.
6. **On stream complete:** Dispatch `COMPLETE_AI_STREAM`.
7. **On error:** Dispatch `REJECT_AI_ACTION` to clean up state.
