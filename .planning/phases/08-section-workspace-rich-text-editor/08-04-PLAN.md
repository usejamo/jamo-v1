**File 1: `src/hooks/useComplianceCheck.ts`**

Two-pass compliance check that fires on accept (D-13). Pass 1 is rule-based (free, instant). Pass 2 is Haiku call (only if rules pass) (D-14).

```typescript
import { useCallback } from 'react'
import { supabase } from '../lib/supabase'

```

Export:
```typescript
export function useComplianceCheck(proposalId: string, orgId: string)
```

Returns: `{ checkCompliance: (sectionKey: string, content: string) => Promise<void> }`

`checkCompliance` implementation:

**Pass 1 — Rule-based checks (instant, free):**
```typescript
const ruleFlags: ComplianceFlag[] = []
const textContent = content.replace(/<[^>]*>/g, '') // strip HTML
const wordCount = textContent.trim().split(/\s+/).length

// 1. Minimum word count check
if (wordCount < 50) {
  ruleFlags.push({ id: crypto.randomUUID(), section_key: sectionKey, type: 'warning', message: `Section has only ${wordCount} words — consider expanding`, source: 'rule' })
}

// 2. Placeholder detection
const placeholderMatches = content.match(/\[PLACEHOLDER:.*?\]/g)
if (placeholderMatches) {
  placeholderMatches.forEach(match => {
    ruleFlags.push({ id: crypto.randomUUID(), section_key: sectionKey, type: 'warning', message: `Unfilled placeholder: ${match}`, source: 'rule' })
  })
}

// 3. Required heading check (section-type specific)
// Derive from SECTION_NAMES: e.g. safety_reporting should mention "adverse events" or "safety reporting"
// study_understanding should reference the therapeutic area
// These are lightweight keyword checks, not semantic analysis
```

If rule-based flags found: dispatch `SET_COMPLIANCE_FLAGS` immediately with rule flags. Do NOT call Haiku.

**Pass 2 — Haiku compliance call (only if rules pass):**
If no rule flags: dispatch `SET_COMPLIANCE_CHECKING` with `checking: true`. Call Haiku via `supabase.functions.invoke`:

```typescript
dispatch({ type: 'SET_COMPLIANCE_CHECKING', payload: { section_key: sectionKey, checking: true } })

const { data, error } = await supabase.functions.invoke('compliance-check', {
  body: {
    section_key: sectionKey,
    content: textContent,
    proposal_id: proposalId,
  },
})

dispatch({ type: 'SET_COMPLIANCE_CHECKING', payload: { section_key: sectionKey, checking: false } })
```

NOTE: The compliance-check edge function does not need to be created in this phase. Instead, use a **direct Anthropic API call from the client via `supabase.functions.invoke`** pattern — create a lightweight edge function `compliance-check` that:
- Receives `{ section_key, content, proposal_id }`
- Calls Anthropic API with `claude-haiku-4-5-20251001` model
- System prompt: narrow, structured, section-type-specific requirements (derive from `cro-proposal-generator.js` section definitions + ICH E6/FDA guidance). For example for `safety_reporting`: "Check if this safety reporting section addresses: adverse event reporting procedures, SAE notification timelines, safety monitoring committee, DSMB provisions. Return JSON: { compliant: boolean, flags: string[] }"
- Returns `{ compliant: boolean, flags: string[] }`

Parse response: map each flag string to a `ComplianceFlag` with `type: 'fail'` if `!compliant`, else `type: 'warning'`, `source: 'haiku'`.

Dispatch `SET_COMPLIANCE_FLAGS` with combined flags.

On network error: dispatch flags with single flag `{ type: 'warning', message: 'Compliance check unavailable. Review manually before submitting.', source: 'rule' }`.
