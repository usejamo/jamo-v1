## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, cursor gaps |
| sm | 8px | Button internal padding, compact list items |
| md | 16px | Section card internal padding, toolbar item gaps |
| lg | 24px | Section card vertical gap, panel header padding |
| xl | 32px | Editor workspace lateral padding |
| 2xl | 48px | Major layout breaks (nav-to-editor gap) |
| 3xl | 64px | Page-level vertical rhythm |

Exceptions:
- Touch targets on action toolbar buttons: minimum 44px height (established in `SectionStreamCard.tsx` — `min-h-[44px]`).
- Version history overlay backdrop: full viewport coverage (no spacing token applies).

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body (editor prose) | 16px | 400 (regular) | 1.6 |
| Label / UI chrome | 14px | 400 (regular) | 1.5 |
| Section heading | 16px | 600 (semibold) | 1.3 |
| Toolbar / badge text | 12px | 600 (semibold) | 1 (single line) |

Source: Established in `SectionStreamCard.tsx` (`text-sm text-gray-700 leading-relaxed` for content, `text-base font-semibold` for section headings, `text-xs font-semibold` for badges).

Notes:
- TipTap editor prose uses 16px body at weight 400, line-height 1.6 — matches existing read-only renderer.
- Compliance flag text in the editor margin: 12px, weight 600, color `amber-700` (warning) or `red-600` (fail).
- Version history diff labels: 12px, weight 600.
- Do not introduce a 5th type size.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F9FAFB` (gray-50) | Page background, editor content area background |
| Secondary (30%) | `#FFFFFF` (white) + `#F3F4F6` (gray-100) | Section cards (`bg-white border border-gray-200`), toolbar surfaces, nav panel |
| Accent (10%) | `#E8524A` (jamo-500) | Primary CTAs only |
| Destructive | `#DC2626` (red-600) | Reject action, error state text, discard version |

Source: `src/index.css` (`--color-jamo: #E8524A`, `background-color: #F9FAFB`) and existing component classes.

Accent (`#E8524A`) reserved for:
- "Accept" button on AI action previews (Expand, Condense, Rewrite diff).
- "Generate" / "Regenerate" primary button per section.
- Active section indicator in the left nav (left border or dot).

Do NOT use accent on: lock/unlock toggle, version history button, toolbar secondary actions (Edit, Decline, Close). Those use gray scale only.

Semantic color usage:
- `amber-100 / amber-800`: Compliance warning flags (rule-based pass but Haiku flag), placeholder highlights — consistent with existing `SectionStreamCard` `[PLACEHOLDER]` highlighting.
- `red-50 / red-600`: Compliance fail flags (Haiku returns `compliant: false`), error states.
- `green-50 / green-700`: Accept animation flash on AI action accept — consistent with existing `SuggestedChange.tsx` (`#dcfce7` on accepting state).
- `blue-50 / blue-300`: Preview area border for AI action inline previews — consistent with `SuggestedChange.tsx` (`border-blue-300`, `bg-blue-50`).
- `purple-50 / purple-600`: Wave 3 sections — consistent with `SectionStreamCard` wave badge config.

## Component Inventory

Primary focal point: the center `SectionEditorBlock` content area with its fixed toolbar header. The left nav is secondary orientation.

New components introduced in this phase:

| Component | Description |
|-----------|-------------|
| `SectionWorkspace` | Top-level wrapper replacing `ProposalDraftRenderer`. Three-column: left nav, center editor, right (reserved for Phase 9 chat). |
| `SectionEditorBlock` | Single section: TipTap editor instance + toolbar header + status badge + lock toggle. |

| `AIActionPreview` | Inline preview area below the TipTap editor. Reuses `SuggestedChange` accept/reject pattern. Used for Expand and Condense. |

| `ComplianceFlag` | Inline margin annotation. Amber (warning) or red (fail). Renders inside TipTap editor margin or below section card. |
| `DraftNav` | Reused from `ProposalDraftRenderer`. Extend with status dot per section (complete / needs-review / missing). |
| `ConsistencyCheckBanner` | Full-width banner appearing after all sections complete. Lists cross-section flags. Dismissible. |

Reused components:
- `SuggestedChange.tsx` — accept/reject interaction pattern and animation. Reference for AIActionPreview and RewriteDiffView.
- `StatusBadge.tsx` — section status chips.
- `DraftNav` (inner component of `ProposalDraftRenderer.tsx`) — left nav panel.

## Interaction Contracts

### Section action toolbar placement
- Fixed header at the top of each `SectionEditorBlock` card (not floating). Cards are individually scrollable within the workspace.
- Toolbar buttons: Generate, Regenerate, Expand, Condense, Rewrite. Secondary: Lock/Unlock toggle (icon only), History (icon only).
- All toolbar buttons 44px minimum touch target height.
- Locked sections: all toolbar action buttons disabled (visually muted, `opacity-40 cursor-not-allowed`). Lock toggle remains active.

### AI action preview flow (Expand / Condense)
- Preview renders inline below the TipTap editor within the same section card.
- Preview area: `bg-blue-50 border border-blue-100 rounded-lg` — consistent with `SuggestedChange.tsx`.
- Accept button: `bg-jamo-500 hover:bg-jamo-600 text-white` — consistent with `SuggestedChange.tsx`.
- Decline button: gray text, no background, right-aligned — consistent with `SuggestedChange.tsx`.
- On accept: green flash animation (`#dcfce7` background) then content loads into TipTap — consistent with `SuggestedChange.tsx` accepting state.
- Streaming content shows animated cursor `|` (animate-pulse) — consistent with `SectionStreamCard.tsx` generating state.

### Rewrite diff view (higher friction — D-04)
- Two-column layout: left "Before", right "After" (streaming).
- Explicit "Apply Rewrite" button (not "Accept" — different label to signal higher commitment).
- "Discard" button in red-600 (destructive action).
- Overlay or inline expansion of the section card — does not shift adjacent sections.

### Version history overlay (D-09)
- Triggered by History icon button in section toolbar.
- Fixed position, `z-50`, semi-transparent backdrop `bg-black/40`.
- Panel slides in from right: framer-motion `x: 320 → 0, opacity: 0 → 1`.
- Panel width: 320px on desktop.
- Entry list: each row shows action label + timestamp (e.g. "Before Rewrite — Mar 26, 2:14 pm").
- Selected entry shows diff against live section content (not raw text).
- "Restore" button per entry: `bg-jamo-500 text-white` — same accept visual as AI action preview.
- Close: X button top-right, or click outside backdrop.

### Lock/unlock toggle
- Icon-only button (lock/unlock heroicon SVG).
- Locked state: `text-amber-600 bg-amber-50` badge or icon tint.
- Unlocked state: `text-gray-400` icon.
- Clicking lock on a section with unsaved changes: no confirmation required (lock is non-destructive, just prevents further edits until unlocked).

### Compliance flags (D-13 / D-14 / D-15)
- Fire on accept, not on stream complete.
- Rule-based (instant, free): render immediately after accept.
- Haiku call (async): spinner indicator in margin for ≤2s while call is in-flight.
- Flag placement: inline below the section card footer (not inside TipTap — avoids editor node complexity).
- Flag chip: `bg-amber-100 text-amber-800 text-xs font-semibold rounded px-2 py-0.5` for warnings; `bg-red-100 text-red-700` for fails.
- Placeholder detection flags: `bg-amber-100 text-amber-800` — consistent with `SectionStreamCard` `[PLACEHOLDER]` mark color.

### Consistency check banner (D-12)
- Auto-triggers after all sections complete generation.
- Full-width banner above the section list: `bg-amber-50 border border-amber-200 rounded-lg`.
- Heading: "Cross-Section Review" — 14px semibold.
- Body: bulleted list of flags.
- Dismiss button: X icon, gray.

### Autosave indicator
- Small "Saved" text indicator in the section card footer, gray-400, 12px.
- Updates to "Saving..." during debounce window (1500ms debounce — Claude's discretion default).
- No toast. No modal. Silent save only.

### Section nav status dots (extending DraftNav)
- Three states per section: `complete` (green-500 dot), `needs-review` (amber-500 dot), `missing` (gray-300 dot).
- Active section: left border `border-l-2 border-jamo-500` on nav item.

## Animation Contract

All animations use framer-motion (already installed, used in `SuggestedChange.tsx` and `ProposalDraftRenderer.tsx`).

| Element | Animation |
|---------|-----------|
| AI preview appear | `opacity: 0→1, y: -6→0`, duration 250ms, delay 500ms (matches `SuggestedChange.tsx`) |
| AI preview dismiss | `opacity: 1→0`, duration 250ms |
| Accept flash | `backgroundColor: white → #dcfce7`, duration 400ms (matches `SuggestedChange.tsx`) |
| Version history overlay enter | `x: 320→0, opacity: 0→1`, duration 250ms ease |
| Version history overlay exit | `x: 0→320, opacity: 1→0`, duration 200ms ease |
| Backdrop | `opacity: 0→1`, duration 200ms |
| Consistency banner enter | `opacity: 0→1, y: -8→0`, duration 300ms |
| Streaming cursor | `animate-pulse` class (matches `SectionStreamCard.tsx`) |
