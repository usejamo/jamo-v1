  return (
    <svg className="w-5 h-5" fill="none" stroke="currentColor" viewBox="0 0 24 24">
      <path strokeLinecap="round" strokeLinejoin="round" strokeWidth={2} d="M17 16l4-4m0 0l-4-4m4 4H7m6 4v1a3 3 0 01-3 3H6a3 3 0 01-3-3V7a3 3 0 013-3h4a3 3 0 013 3v1" />
    </svg>

**Handler:**
```typescript
import { useAuth } from '../context/AuthContext'
import { useNavigate } from 'react-router-dom'

// Inside Sidebar component:
const { signOut } = useAuth()
const navigate = useNavigate()

async function handleSignOut() {
  const { error } = await signOut()
  if (!error) {
    navigate('/login')
  } else {
    console.error('Sign out failed:', error)
  }
}
```

**Why this approach:**
- Button (not NavLink) — logout is an action, not navigation
- `text-left` ensures text aligns with nav items
- Navigate to /login after signOut (ProtectedRoute will also redirect, but explicit is clearer)
- Error logged to console (no toast needed — sign out rarely fails)

**DO NOT:**
- Do NOT add confirmation dialog ("Are you sure?") — keep it simple for MVP
- Do NOT add loading spinner — sign out is fast
- Do NOT change existing nav structure
