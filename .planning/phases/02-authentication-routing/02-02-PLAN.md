  <action>
Create `src/components/ProtectedRoute.tsx` that wraps protected routes and redirects unauthenticated users to /login.

**Pattern (React Router v7):**
```typescript
import { Navigate, Outlet } from 'react-router-dom'
import { useAuth } from '../context/AuthContext'

export function ProtectedRoute() {
  const { session, loading } = useAuth()

  // Show nothing while checking auth state
  if (loading) {
    return (
      <div className="min-h-screen flex items-center justify-center">
        <div className="text-gray-500">Loading...</div>
      </div>
    )
  }

  // Redirect to login if no session
  if (!session) {
    return <Navigate to="/login" replace />
  }

  // Render child routes if authenticated
  return <Outlet />
}
```

**Why this works:**
- `<Outlet />` renders nested routes (Dashboard, ProposalsList, etc.)
- `loading` check prevents flash of login page during initial session load
- `replace` in Navigate prevents back-button loop
- Simple, minimal, follows React Router v7 patterns

**DO NOT:**
- Do NOT add role checking (that's Plan 03 / future)
- Do NOT add org checking (RLS handles that at DB level)
- Do NOT add complex loading UI (keep it simple — 1-2 second flash at most)
