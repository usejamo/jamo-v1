    Create src/components/FileUpload.tsx:

    **Component props:**
    - `proposalId: string` — proposal these files belong to
    - `onUploadComplete?: (documentId: string) => void` — callback after successful upload

    **Visual design (match Jamo design system):**
    - Drag-and-drop zone with dashed border (border-dashed border-2 border-gray-300)
    - Hover state: border-jamo-500 bg-jamo-50
    - File icon (cloud upload icon from lucide-react or similar)
    - "Drag files here or click to browse" text
    - File type hint: "Supports: PDF, DOCX, XLSX, TXT (max 50MB)"
    - Multiple file upload support

    **Upload flow:**
    1. Get org_id from `useAuth().profile.org_id` (throw if not authenticated)
    2. For each file:
       a. Validate file type (accept: .pdf, .docx, .doc, .xlsx, .xls, .txt)
       b. Validate size (max 50MB per Supabase limit)
       c. Generate storage path: `${org_id}/${proposalId}/${file.name}`
       d. Upload to Supabase Storage:
          ```typescript
          const { data: uploadData, error: uploadError } = await supabase.storage
            .from('documents')
            .upload(storagePath, file, {
              cacheControl: '3600',
              upsert: false
            })
          ```
       e. On success, insert row into proposal_documents:
          ```typescript
          const { data: docData, error: docError } = await supabase
            .from('proposal_documents')
            .insert({
              org_id: profile.org_id,
              proposal_id: proposalId,
              uploaded_by: user.id,
              name: file.name,
              storage_path: uploadData.path,
              mime_type: file.type,
              size_bytes: file.size,
              parse_status: 'pending'
            })
            .select()
            .single()
          ```
       f. Call onUploadComplete(docData.id) if provided

    **Per-file state tracking:**
    - Use useState to track upload progress per file: `{ file: File, status: 'uploading' | 'complete' | 'error', error?: string }`
    - Display per-file status indicators in UI (uploading spinner, green check, red X)

    **Error handling:**
    - Storage upload error → set status 'error', display user-friendly message
    - Database insert error → attempt Storage cleanup (supabase.storage.from('documents').remove([path]))
    - File type validation error → show "Unsupported file type" message
    - Size validation error → show "File too large (max 50MB)" message

    **Tailwind styling:**
    - Use existing Jamo design patterns from src/components/ProposalEditorModal.tsx
    - Consistent spacing (px-6, py-4, gap-4)
    - Focus states (focus:ring-2 focus:ring-jamo-200)
