<decisions_made>

- **vitest 4.0.18 → 4.0.4**: Downgraded due to known memory leak regression in 4.0.18 (GitHub issue #9560). Tamagui monorepo documented 4.0.4 as the working version.
- **jsdom → happy-dom**: Switched environment to reduce memory footprint per worker.
- **pool: forks, singleFork: true**: Forces all tests into a single fork process to avoid spawning multiple 4GB workers.
- **execArgv --max-old-space-size=4096**: Passes heap limit to the fork process.
- **Inline mock objects**: Replaced `() => import('../../test/mocks/supabase')` (Promise-returning factory) with inline `{ supabase: { from: vi.fn()... } }` objects. The Promise pattern caused vitest to resolve the full supabase-js module before the mock intercepted it.
- **Env in .env not .env.local**: Project uses `.env` (not `.env.local`) for Supabase credentials.
- **VITE_SUPABASE_PUBLISHABLE_KEY not VITE_SUPABASE_ANON_KEY**: Actual key name differs from plan template.

</decisions_made>
