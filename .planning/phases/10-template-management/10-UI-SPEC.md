## Component Inventory

### 1. TemplateSelector (Step 4 Wizard — above ContextSummary)

**Layout:** Collapsible section with heading "Choose a template (optional)". Expanded by default. Card grid: `grid grid-cols-1 gap-3` on mobile, `grid-cols-2 gap-3` on md+.

**Template card anatomy:**
- Border: `border rounded-lg p-4 cursor-pointer transition-colors`
- Default state: `bg-white border-gray-200 hover:border-jamo-200`
- Selected state: `bg-jamo-50 border-jamo-300 ring-2 ring-jamo-200`
- Disabled state (parse_status !== 'ready'): `bg-gray-50 border-gray-200 opacity-50 cursor-not-allowed`
- Card content top row: template name (`text-sm font-semibold text-gray-900`) + source badge (right-aligned)
- Card content second row: description (`text-sm text-gray-500 mt-1 line-clamp-2`)
- Source badge "Pre-built": `text-xs font-normal text-gray-500 bg-gray-100 px-2 py-0.5 rounded-full`
- Source badge "Your template": `text-xs font-normal text-jamo-600 bg-jamo-50 px-2 py-0.5 rounded-full`

**Separator:** Between pre-built and org-uploaded templates — `<hr className="my-2 border-gray-200">` with label `<span className="text-xs text-gray-400">Your organization's templates</span>` centered above it.

**Deselect affordance:** Clicking a selected card deselects it (dispatches `SET_TEMPLATE` with `null`).

**ContextSummary line when no template selected:** "No template — using standard structure" in `text-gray-400 italic`.

---

### 2. TemplatesTab (Settings page)

**Location:** New sub-tab added to `Settings.tsx` `SUB_TABS` array as `'Templates'`. Tab only visible to users where `role === 'admin'` — non-admins do not see the tab at all.

**Tab layout — three stacked sections:**

**Section A: Upload zone**
- Heading: "Upload a template" (`text-base font-semibold text-gray-900`)
- Subtext: "DOCX or PDF only. Uploaded templates are private to your organization." (`text-sm text-gray-500`)
- Reuse `FileUpload.tsx` restricted to `.docx,.pdf` accept types
- Upload zone minimum height: 120px
- Post-upload: immediately adds a row to Section B with `parse_status: 'pending'`

**Section B: Template list**
- Heading: "All templates" (`text-base font-semibold text-gray-900`)
- Pre-built templates listed first, then org-uploaded (matches D-03)
- Each row: `flex items-center justify-between py-3 border-b border-gray-100`
  - Left: template name (`text-sm font-semibold text-gray-900`) + description (`text-xs text-gray-500 mt-0.5`)
  - Right: status badge + (admin-only) delete icon button
- Status badge uses `StatusBadge.tsx`:
  - `pending` / `extracting`: blue — "Processing"
  - `ready`: green — "Ready"
  - `error`: red — "Error"
- Pre-built rows: no delete button (source = 'prebuilt' → delete hidden)
- Delete button: `text-gray-400 hover:text-red-600` icon-only, 44px touch target, `aria-label="Delete template"`

**Section C: Section disclosure per template**
- Placed inline below each template row when row is expanded
- Trigger: `<button>` styled as `text-sm text-gray-500 hover:text-gray-700` — "View detected sections (N) ▾"
- Collapsed by default; toggle via `useState<boolean>`
- Expanded content: read-only `<ol className="mt-2 ml-4 space-y-1 list-decimal text-sm text-gray-600">`
- Low-confidence warning (when `low_confidence = true`): amber inline alert above section list
  - `bg-amber-50 border border-amber-200 rounded-md px-3 py-2 text-sm text-amber-700`
  - Copy: "Fewer sections than expected were detected — review before using this template."
  - No icon required; no blocking action.

---

### 3. Delete Confirmation Dialog

**Trigger:** Admin clicks delete icon on an org-uploaded template row.

**Pattern:** Inline modal/dialog (reuse existing dialog pattern in the project — check for existing `<dialog>` or overlay component).

**Content:**
- Heading: "Delete template?" (`text-base font-semibold`)
- Body: "This template may have been used in existing proposals. Deleting it will not affect those proposals, but it will no longer be available for selection." (`text-sm text-gray-600`)
- Actions: "Cancel" (secondary button) + "Delete" (`bg-red-600 hover:bg-red-700 text-white` button)
- Cancel is the leftmost / default-focused action.

## Accessibility

- All icon-only buttons must have `aria-label`
- Delete button: `aria-label="Delete [template name]"` (include name for screen readers)
- Template cards: `role="radio"` within a `role="radiogroup"` (single-select pattern); `aria-checked` reflects selection state
- Section disclosure: `aria-expanded` on the toggle button
- Low-confidence warning: `role="alert"` so it is announced on appearance
- Status badge for processing templates: `aria-live="polite"` on the status cell to announce transitions
