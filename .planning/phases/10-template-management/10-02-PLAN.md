Create `supabase/functions/template-extract/index.ts` modeled on `extract-document/index.ts` but storing results in `template_sections` (per D-17, RESEARCH.md Pattern 6).

**Imports** (match extract-document pattern exactly):
```typescript
import { serve } from 'https://deno.land/std@0.168.0/http/server.ts'
import { createClient } from 'supabase'
```

**Request body:** `{ templateId: string }`

**Flow:**
1. Create Supabase admin client (service_role key from env, same as extract-document)
2. Validate `templateId` is a valid UUID
3. Fetch template row: `supabase.from('templates').select('*').eq('id', templateId).single()`
4. Verify template exists and has `file_path`
5. Update parse_status to `'extracting'`
6. Download file from Supabase Storage: `supabase.storage.from('documents').download(template.file_path)`
7. Determine file type from file_path extension (.docx or .pdf)
8. Extract content:
   - **DOCX**: Use `mammoth.convertToHtml({ buffer })` (NOT `extractRawText` — need heading tags for section detection per RESEARCH.md Pitfall 1). Import as `const mammoth = await import('mammoth')`
   - **PDF**: Use the same pdfjs import pattern from extract-document. Extract text, then use heading heuristic.
9. **Parse sections from extracted content:**
   - For DOCX HTML output: Parse `<h1>`, `<h2>`, `<h3>` tags using regex: `/<h[1-3][^>]*>(.*?)<\/h[1-3]>/gi`. Each match becomes a section name. Assign position sequentially starting at 1.
   - For PDF text: Lines that are ALL-CAPS or lines shorter than 80 chars ending without punctuation are candidate headings (per RESEARCH.md Pattern 6).
   - Assign `role = null` for all detected sections (uploaded templates have org-specific names that may not map to standard roles).
   - Set section `description = null` (read-only, user cannot edit per D-21).
10. **Low-confidence detection (per RESEARCH.md Pattern 5):**
    - Set `low_confidence = true` if heading count < 3 OR total word count < 200
11. **Bulk-insert template_sections:**
    ```typescript
    const sectionRows = sections.map((s, i) => ({
      template_id: templateId,
      name: s.name,
      role: null,
      description: null,
      position: i + 1,
      org_id: template.org_id,  // denormalized from parent
    }))
    await supabase.from('template_sections').insert(sectionRows)
    ```
12. **Update template row:**
    ```typescript
    await supabase.from('templates').update({
      parse_status: 'ready',
      low_confidence: isLowConfidence,
    }).eq('id', templateId)
    ```
13. **Error handling:** Wrap in try/catch. On error, update `parse_status = 'error'` on the template row. Return 500 with error message.
14. Return 200 with `{ success: true, sectionCount: sections.length }`

**CORS headers:** Include the same CORS headers as extract-document (Access-Control-Allow-Origin: *, etc.)
